Installation, Configuration, Documentation and Vulnerability Analysis
# Task 3: SQL Injection Demonstration Using DVWA

## Objective

The objective of this project was to install and configure Damn Vulnerable Web Application (DVWA) on a local Kali Linux environment and study a SQL Injection vulnerability using DVWA's intentionally vulnerable training environment.

## Tools Used

* Kali Linux
* Apache2
* PHP
* MariaDB
* DVWA (Damn Vulnerable Web Application)

## Project Overview

The project involved configuring a complete web application environment, creating the required database, installing DVWA, and preparing the application for security testing.

The SQL Injection module was examined using DVWA's **Low** security level to understand how insecure database queries can allow unexpected behavior when user input is not properly validated.

## Skills Demonstrated

* Linux administration
* Apache web server configuration
* PHP configuration
* MariaDB database administration
* DVWA deployment
* Secure documentation
* Vulnerability analysis

## Files Included

* installation-guide.md
* findings.md
* sql_injection_exploit.sh
* screenshots/

## Learning Outcome

This project demonstrates how insecure database query construction can introduce SQL Injection vulnerabilities and highlights the importance of secure coding practices such as parameterized queries, prepared statements, proper input validation, and least-privilege database permissions.
